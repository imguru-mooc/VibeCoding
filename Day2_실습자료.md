## 오전 1교시 (10:00~11:00): 프롬프트의 기초 이해

### 프롬프트란 무엇인가?

프롬프트(Prompt)는 AI에게 전달하는 **지시문, 질문, 맥락 정보의 총합**입니다.
같은 AI라도 프롬프트에 따라 결과물 품질이 **10배 이상** 달라집니다.

프롬프트를 이해하기 위해 비유를 하나 들어보겠습니다.

**식당 주문에 비유하면:**
- 나쁜 주문: "맛있는 거 주세요" → 뭘 줄지 모름
- 보통 주문: "파스타 주세요" → 어떤 파스타? 맵기는?
- 좋은 주문: "크림 파스타, 면은 알덴테로, 매운맛 빼주시고, 15분 안에 부탁합니다" → 정확히 원하는 것이 나옴

AI에게도 똑같습니다. **구체적으로 시킬수록 정확한 결과**가 나옵니다.

### 프롬프트의 5가지 핵심 구성 요소

좋은 프롬프트에는 다음 5가지가 들어있습니다.

| 순서 | 요소 | 설명 | 예시 |
|------|------|------|------|
| 1 | **역할 (Role)** | AI에게 전문가 역할을 부여 | "너는 10년 경력의 마케팅 전문가야" |
| 2 | **맥락 (Context)** | 배경 정보, 상황, 목적 설명 | "B2B SaaS 스타트업의 신규 서비스 론칭을 앞두고" |
| 3 | **지시 (Instruction)** | 구체적으로 해야 할 일 | "마케팅 전략 보고서를 작성해줘" |
| 4 | **세부사항 (Specifics)** | 조건, 제약, 범위 | "예산 500만원, 기간 3개월, 타겟은 IT 담당자" |
| 5 | **출력 형식 (Format)** | 원하는 결과물 형태 | "표 형식, A4 2장, 핵심 볼드 처리" |

이 5가지를 쉽게 기억하는 방법이 바로 **CRISP 템플릿**입니다.

## 오전 2교시 (11:00~12:00): CRISP 템플릿 마스터

### CRISP 템플릿이란?

CRISP는 효과적인 프롬프트를 만드는 **만능 공식**입니다.

| 알파벳 | 의미 | 핵심 질문 |
|--------|------|---------|
| **C** | Context (맥락) | 어떤 상황인가? |
| **R** | Role (역할) | AI가 어떤 전문가 역할을 해야 하는가? |
| **I** | Instruction (지시) | 구체적으로 무엇을 해야 하는가? |
| **S** | Specifics (세부사항) | 어떤 조건과 제약이 있는가? |
| **P** | Preference (선호) | 결과물을 어떤 형식으로 원하는가? |

**CRISP를 모두 채우면 반드시 좋은 결과가 나옵니다.**
단, 매번 5가지를 다 넣을 필요는 없습니다. **최소 C + I + P (맥락 + 지시 + 형식)** 만 넣어도 결과가 크게 달라집니다.

## 오후 1교시 (13:00~14:30): 고급 프롬프트 기법 4가지

### 기법 1: Zero-Shot vs Few-Shot

#### 개념 설명

**Zero-Shot**은 예시 없이 바로 시키는 방법입니다.
"이 문장의 감성을 분석해줘" → AI가 알아서 판단

**Few-Shot**은 예시를 먼저 보여주고 시키는 방법입니다.
"이런 게 긍정이고, 이런 게 부정이야. 그러면 이건?" → AI가 패턴을 학습

**언제 Few-Shot을 쓰는가?**
- 특정한 형식이나 스타일로 결과를 받고 싶을 때
- 분류/평가 기준을 AI에게 알려줘야 할 때
- Zero-Shot 결과가 만족스럽지 않을 때

---

### 실습 3: Zero-Shot vs Few-Shot 비교 체험 (13:10~13:30)

#### 비교 실험 1: 고객 리뷰 감성 분석

**Zero-Shot 프롬프트:**
```
다음 고객 리뷰의 감성을 분석해줘 (긍정/부정/중립):

1. "배송이 빨라서 너무 좋았어요"
2. "제품은 괜찮은데 포장이 좀 아쉬웠습니다"
3. "다시는 안 삽니다. 완전 실망"
4. "가격 대비 그냥 무난해요"
5. "친구한테 추천했더니 같이 재구매했어요!"
```

→ AI에 입력하고 결과를 기록하세요.

**Few-Shot 프롬프트:**
```
다음 고객 리뷰의 감성을 분석해줘.
아래 예시를 참고하여 동일한 형식으로 분석해줘:

[예시]
리뷰: "정말 만족합니다. 다음에도 꼭 구매할게요"
감성: 긍정 (★★★★★)
근거: "정말 만족", "꼭 구매" 등 강한 긍정 표현

리뷰: "포장이 찢어져서 왔어요. 교환 요청합니다"
감성: 부정 (★★☆☆☆)
근거: 제품 손상, 교환 요청 = 불만족

리뷰: "보통이에요. 특별한 건 없네요"
감성: 중립 (★★★☆☆)
근거: "보통", "특별한 건 없다" = 중간 평가

[분석 대상]
1. "배송이 빨라서 너무 좋았어요"
2. "제품은 괜찮은데 포장이 좀 아쉬웠습니다"
3. "다시는 안 삽니다. 완전 실망"
4. "가격 대비 그냥 무난해요"
5. "친구한테 추천했더니 같이 재구매했어요!"
```

**비교 기록:**

| 평가 항목 | Zero-Shot | Few-Shot |
|----------|-----------|----------|
| 분류 정확도 (1~5) | | |
| 분석 형식 일관성 (1~5) | | |
| 근거 설명 충실도 (1~5) | | |
| 실무 활용 가능성 (1~5) | | |

#### 비교 실험 2: 회의록 요약

**Zero-Shot:**
```

### 기법 2: Chain-of-Thought (CoT) — 단계적 사고 유도

#### 개념 설명

Chain-of-Thought는 AI에게 **"단계별로 생각해"**라고 시키는 기법입니다.

**왜 효과적인가?**
- 복잡한 문제를 작은 단계로 나누면 AI가 더 정확하게 답변합니다
- 각 단계의 논리를 확인할 수 있어서 오류를 잡기 쉽습니다
- 피상적인 답변 대신 체계적이고 깊이 있는 분석을 받을 수 있습니다

**언제 쓰는가?**
- 분석 보고서, 전략 수립, 원인 분석 등 **복잡한 사고가 필요한 업무**
- AI가 일반적이고 피상적인 답변만 줄 때
- "좀 더 깊이 있게 분석해줘"로 안 될 때

### 실습 4: Chain-of-Thought 따라하기 (13:30~14:00)

```

→ 결과를 기록하세요. 아마 일반적인 답변이 나올 겁니다.

**CoT 프롬프트 (다음으로 입력):**
```

단계별로 분석해줘:

Step 1) 가능한 외부 원인 5가지
- 각 원인의 발생 가능성을 상/중/하로 평가
- 근거도 함께 제시

- 확인 방법 제시 (어떤 데이터를 보면 알 수 있는지)

Step 3) 가장 유력한 원인 Top 3 선정
- 선정 이유를 논리적으로 설명

Step 4) 각 원인별 구체적 대응 방안
- 단기(1주), 중기(1개월), 장기(3개월)로 구분
- 예상 비용과 효과도 함께 제시

- 효과(높음/낮음) x 실행 난이도(쉬움/어려움) 2x2 매트릭스로 정리
- 가장 먼저 실행할 액션 3가지 선정

표와 함께 정리해줘.
```

| 실행 가능성 (1~5) | | |
| 논리적 구조 (1~5) | | |
| 경영진 보고 적합성 (1~5) | | |

#### 따라하기 B: 신규 사업 아이디어 분석

```
우리 팀에서 "직장인 대상 AI 활용 교육 서비스" 사업을 검토 중이야.

단계별로 분석해줘:

Step 1) 시장 분석
- 시장 규모와 성장률 추정
- 주요 경쟁사 3~5개와 그들의 강점/약점

Step 2) 고객 분석
- 타겟 고객 페르소나 3가지 정의
- 각 페르소나의 핵심 니즈와 Pain Point

Step 3) SWOT 분석
- 표 형식으로 강점/약점/기회/위협 정리

Step 4) 수익 모델 설계
- 가능한 수익 모델 3가지 비교
- 각 모델의 예상 매출과 손익분기점

Step 5) 리스크 분석

- 리스크 영향도/발생 확률 매트릭스

Step 6) Go/No-Go 결론
- 위 분석을 종합한 추천 의견
- 조건부 Go라면 필요한 전제조건 제시
```

**결과 평가:**
- [ ] 각 단계가 논리적으로 연결되는가?
- [ ] 구체적인 수치/근거가 포함되어 있는가?
- [ ] 실제 경영진 보고에 쓸 수 있는 수준인가?
- [ ] 추가로 보완이 필요한 단계는 어디인가?

## 오후 2교시 (14:30~15:30): 프롬프트 체이닝 실전

### 프롬프트 체이닝이란?

하나의 복잡한 업무를 **여러 번의 대화로 나누어** 점진적으로 완성하는 기법입니다.

**왜 체이닝을 쓰는가?**
- 한 번에 모든 것을 시키면 AI가 각 부분의 품질을 균등하게 나누게 됨
- 단계별로 시키면 각 단계에 집중하여 더 높은 품질 확보
- 중간 결과를 확인하고 방향 수정이 가능

**체이닝의 기본 패턴:**
```
1단계: 아이디어 발산 (브레인스토밍)
    ↓
2단계: 평가 및 선별 (기준 제시 후 필터링)
    ↓
3단계: 구체화 (선택된 아이디어 상세 전개)
    ↓
4단계: 실행 계획 (타임라인, 담당자, 예산)
    ↓
5단계: 최종 문서화 (보고서, 제안서 형태로 정리)
```

### 실습 7: 프롬프트 체이닝 풀코스 따라하기 (14:40~15:20)

**1단계 — 브레인스토밍:**

- 비용이 거의 들지 않는 방법 (무료 AI 도구 활용)

- 각 아이디어에 예상 시간 절감 효과도 표시해줘

**2단계 — 평가 및 선별:**

1) 시간 절감 효과 (높을수록 좋음)
2) 적용 난이도 (낮을수록 좋음)
3) 팀원 수용도 (높을수록 좋음)

```

→ Top 3가 선정되었나요? 납득이 되나요?

**3단계 — 구체화:**
```
위에서 선정된 Top 3 아이디어를 각각 구체화해줘.

각 아이디어별로:
1) 현재 업무 프로세스 (As-Is)
2) AI 적용 후 프로세스 (To-Be)
3) 필요한 AI 도구와 설정 방법
